# QA — hotfix v2.0.2 scope conflict

**Fecha**: 2026-07-22
**Versión testeada**: 2.0.2-dev
**Verdict**: **APPROVED** ✅

## Resumen

Hotfix v2.0.2 corrige el bug crítico que dejaba la app completamente inutilizable (dashboard no respondía a ningún click, ni el X cerrar). Root cause: `src/core/context-awareness.js` declaraba `function shouldEnterDoNotDisturb(...)` a top-level, lo que colisionaba con el `const { shouldEnterDoNotDisturb } = window.ContextAwareness;` de `dashboard-renderer.js` en el mismo scope global del dashboard window (regla strict mode "lexical declaration collision").

Fix: wrappar `context-awareness.js` en IIFE para que las funciones queden privadas y NO contaminen el global. API pública intacta.

## Test 1 — runtime evidence del fix

Simulé el orden de carga del browser (context-awareness.js → dashboard-renderer.js) en un sandbox `new Function()` con `window` mock:

```js
const fs = require('fs');
const contextAwareness = fs.readFileSync('src/core/context-awareness.js', 'utf8');
const dashboardFragment = `
  const { shouldEnterDoNotDisturb, shouldExitDoNotDisturb, computeTypingRate } = window.ContextAwareness;
  return {
    hasEnter: typeof shouldEnterDoNotDisturb === 'function',
    hasExit: typeof shouldExitDoNotDisturb === 'function',
    hasRate: typeof computeTypingRate === 'function',
    enterCall: shouldEnterDoNotDisturb([]),
    exitCall: shouldExitDoNotDisturb([])
  };
`;

const window = {};
const code = contextAwareness + '\n' + dashboardFragment;
const result = new Function('window', code)(window);
console.log(result);
```

**Output**:
```
{
  "hasEnter": true,
  "hasExit": true,
  "hasRate": true,
  "enterCall": false,
  "exitCall": true
}
```

✅ **SINTAXIS_OK**: context-awareness.js + dashboard-renderer.js NO colisionan
✅ **DESTRUCTURE_OK**: las 3 funciones se destructuran correctamente
✅ **SMOKE_OK**: las funciones devuelven los valores esperados (`[]` → enter=false, exit=true)

## Test 2 — antes del fix (sentinel)

Mismo test pero con el archivo ORIGINAL (sin IIFE) habría tirado:
```

```

Esto está verificado automáticamente por el test `BIDIRECCIONAL: sin IIFE, el conflict SE REPRODUCE (sentinel)` en `test/context-awareness-scope-conflict.test.js`.
