## ✨ Features nuevas

### Track A — T7 + T6 (Distribución + Auto-update)

#### T7 — Empaquetado con electron-builder

Por primera vez Mascota Virtual es **instalable en Windows como cualquier
app de escritorio**.

**Artifact**: `MascotaVirtual-Setup-2.0.0.exe` (~80-120 MB)

**Target**: NSIS installer (Windows 10/11, x64)

**Configuración**:
- `appId`: `com.hosneydertmesa.mascotavirtual`
- `productName`: `Mascota Virtual`
- `icon`: `assets/icon.ico` (256x256, generated)
- `asar`: true (empaquetado, ~30% más chico)
- `oneClick: false` (usuario decide dónde instalar)
- `perMachine: false` (per-user, no requiere admin)
- `createDesktopShortcut: true`

**Scripts**:
- `npm run build` → `electron-builder --win --x64` (NSIS)
- `npm run build:dir` → `electron-builder --dir` (sin instalador, para testing rápido)

**Nota importante**: el instalador es **self-signed** (sin cert EV).
Windows SmartScreen va a mostrar "Editor desconocido" la primera vez.
El usuario tiene que hacer click "More info → Run anyway". Esto es
aceptable para early adopters. Cert EV ($300/yr) se difiere a v2.1.0
si la adopción lo justifica.

#### T6 — Auto-update desde GitHub Releases

La app **se actualiza sola** cuando hay una nueva versión publicada
en GitHub Releases. No más pedirle a la gente que baje un .exe nuevo.

**Wire**:
- `autoUpdater.checkForUpdates()` se llama al startup (solo si `app.isPackaged`)
- `setInterval(checkForUpdates, 6h)` con `.unref()` (no bloquea quit)
- `autoUpdater.autoDownload = true` — descarga en background
- `autoUpdater.autoInstallOnAppQuit = true` — instala al cerrar
- `autoUpdater.allowDowngrade = false` — no permite downgrades

**Eventos forwarded al renderer**:
- `update-available` → "Update v2.0.1 downloading..."
- `update-downloaded` → "Update v2.0.1 listo. Se instala al cerrar la app."

**Channel**: GitHub Releases (`HosneydertMesa/mascota-virtual`).
electron-updater verifica checksums automáticamente.

**Release workflow** (`.github/workflows/release.yml`):
- Dispara en `push: tags: ['v*']` o `workflow_dispatch`
- Build en `windows-latest`
- Steps: `npm ci` → `npm run build` → `gh release create ${{ github.ref_name }}`

**Cómo publicar una nueva versión**:
```bash
npm version patch -m "chore(release): v%s"
git push origin main --follow-tags
# GH Actions buildea y crea el release con el .exe
```

#### CSP — Content Security Policy (security audit MEDIUM-1)

Dashboard ahora tiene CSP estricto en `<meta>`:
```html
default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline';
img-src 'self' data:; connect-src 'none'; media-src 'self';
object-src 'none'; base-uri 'none'
```

`connect-src 'none'` bloquea exfil. Todas las llamadas externas
(API de M3, GitHub) van por IPC al main, que valida sender.

### Track B — W1 + W2 (Modos de comportamiento)

#### W1 — Modo compañía silenciosa (visual only)

Toggle que pone la mascota en modo "solo presencia, sin conversación".
Útil cuando estás concentrando y no querés que te hable.

**Comportamiento** cuando `silentMode = true`:
- ❌ No inicia chat proactivamente
- ❌ No reacciona a mood changes (mood se queda donde estaba)
- ❌ No muestra tips de inactividad (A3)
- ❌ No dispara warnings de DND por typing (A4)
- ❌ No muestra daily briefing / evening summary (I7/I8)
- ✅ Animaciones idle SÍ funcionan (respiración, eye tracking, blink)
- ✅ Drag/resize SÍ funcionan

**Toggle**:
- Dashboard Settings → "Modo compañía silenciosa" (checkbox)
- `Cmd/Ctrl+Shift+M` (atajo global, M de "Mute")
- Persiste en `<userData>/pet-config.json`

**Visual**:
- opacity 0.7
- filter: saturate(0.6) brightness(0.95)
- transition 0.4s ease

**Decisión de shortcut**: originalmente el plan era `Cmd/Ctrl+Shift+S`,
pero `Shift+S` ya estaba wireado a `pet-sleep` (batch 0). Se eligió
`Shift+M` para no romper la memoria muscular existente.

#### W2 — Auto-pause en reuniones (.ics local)

Lee un archivo `.ics` local (export de Outlook, Apple Calendar, Google
Calendar) y automáticamente pone a la mascota en "retreat mode" cuando
hay una reunión en los próximos 5 minutos.

**Configuración**:
- Dashboard Settings → "Ruta al archivo .ics" (input + botón "Probar")
- El botón "Probar" parsea y muestra los próximos 3 eventos

**Parser**: `node-ical@0.27.1` (primary) + `parseIcsMinimal` (fallback
con regex propio, cubre 3 formatos iCal). Si `node-ical` falla al cargar
(por ej. en una versión futura con vulns), el fallback toma el control.

**Retreat mode** cuando hay evento en próximos 5 min:
- Equivalente a silent mode + visuals adicionales
- opacity 0.5, scale 0.7, posición esquina
- Speech bubble: `🗓️ {summary} hasta las HH:MM` por 8s
- Cuando termina el evento → vuelve a estado normal

**File watcher**: `fs.watch` sobre el `.ics`. Si el usuario edita el
archivo o el calendar app lo re-exporta, se re-parsea automáticamente.

**Scheduler**: `setInterval(evaluateRetreatState, 30_000).unref()`.
Chequea eventos cada 30s. Primera evaluación a los 2s de startup (para
detectar reuniones ya en curso).

**Path traversal check** en `config:set-calendar-path`:
`if (filePath.includes('..')) throw` — previene que el usuario configure
un path malicioso.

**Persistencia**: en `pet-config.json` (nuevo, FILE_VERSION 1, atomic write).

### Track C — T5 60fps performance pass

Optimizaciones en hot paths del renderer + red de seguridad contra
regresiones.

**Optimizaciones aplicadas** en `src/renderer.js`:

1. **Cache de DOM refs en hot paths**:
   - `catTailEl`, `dogTailEl` — antes `getElementById` cada frame (60Hz)
   - `earLeftEl`, `earRightEl` — antes `querySelector` en `maybeTwitchEar`
   - `pupilEls` (Array) — antes `querySelectorAll` en cada mousemove
   - Función `cachePetElements()` re-cachea después de `loadMascotSVG`
     (que re-injecta `innerHTML` y destruye los nodos)

2. **Loop optimization**:
   - `pupils.forEach(...)` → `for (let i = 0; ...)` en `handlePupilTracking`
   - Micro-opt pero en 100+ mousemove/s suma

**Red de seguridad** — `test/performance-budget.test.js` (nuevo):
- 9 tests que miden tiempo de funciones puras
- Budget 50ms / 1000 iter (generoso: detecta regresiones 10x)
- Resultado: todas < 1ms (1000x+ bajo el budget)

**Funciones medidas**:
- `pomodoro-templates.formatTemplateForDisplay`
- `pomodoro-adaptive.shouldUseLongBreak`
- `pomodoro-streak.computeStreak` (90 días)
- `quick-capture.validateCaptureText`
- `weekly-report.buildWeeklyReport` (7 días)
- `pet-memories.extractPII`
- `silent-mode.isSilentModeActive`
- `auto-updater.isNewerVersion`
- `calendar-service.parseIcsDate`

**Baseline documentado** en `docs/perf/baseline-2026-07-22.md`.
Performance real (60fps) requiere profiling con DevTools en QA.

## 🔧 Decisiones técnicas documentadas

1. **electron-builder 25.x → 26.x** durante el merge. 25.x tenía
   transitive vulns (`tar`/`node-gyp`); 26.x es la línea soportada.
2. **Self-signed installer** (decisión del user). SmartScreen warning
   es OK para early adopters. Cert EV ($300/yr) diferido a v2.1.0.
3. **`Cmd/Ctrl+Shift+M`** en vez de Shift+S (conflicto con pet-sleep).
4. **`node-ical` + parser propio** como fallback. Robustez ante
   vulnerabilidades futuras de la dep.
5. **`pet-config.json`** es nuevo (no existía). Patrón FILE_VERSION +
   atomic write consistente con stores anteriores.
6. **CSP `connect-src 'none'`** — más estricto que el sugerido en el
   audit. No rompe nada porque las llamadas externas van por IPC.
