## Features nuevas

### Track A — I1 + W4 + W5 Pomodoro adaptativo

#### I1 — Pomodoro adaptativo

El timer clásico 25/5 ahora se adapta: después de 4 focus blocks
consecutivos sin break largo, el siguiente break es **largo** (15 min
por default, configurable). El contador se resetea con el break largo,
evitando dobles breaks largos consecutivos.

**Heurística** (en `src/core/pomodoro-adaptive.js`):
- `nextBreakKind({focusBlocksCompleted, lastBreakWasLong, longBreakEvery})`
- Si `focusBlocksCompleted % longBreakEvery === 0` y `!lastBreakWasLong` → `'long'`
- Si no → `'short'`
- Default `longBreakEvery = 4` (configurable)

**IPC**:
- `pomodoro:get-next-break-kind` — el renderer consulta antes de iniciar break

#### W4 — Plantillas de pomodoro

4 plantillas pre-definidas + 1 custom:
- **Classic**: 25 min focus / 5 min break
- **Long focus**: 50 min / 10 min
- **Deep work**: 90 min / 20 min
- **Custom**: focus 5-120, break 1-30, long break 5-60, long every 2-10

**Persistencia**: `<userData>/pomodoro-config.json` (atomic write)

**IPC**:
- `pomodoro:get-config` / `pomodoro:set-config`

#### W5 — Streak tracking

Racha de **días consecutivos con ≥1 focus block completado**. Si un
día no hay focus → racha rota.

**Mensajes motivacionales** (en `src/core/pomodoro-streak.js`):
- 3, 7, 14, 30, 60, 100 días → mensaje con tono cat/dog
- Ej cat 7: "¡7 días seguidos con focus! Miau, qué orgullosa estoy."
- Ej dog 7: "¡7 días seguidos! Guau, qué compañero sos."

**Persistencia**: en `pomodoro-sessions.json` (no archivo nuevo)

**Stats widgets** en dashboard:
- "X focus hoy"
- "Y esta semana"
- "Z días de racha"

**IPC**:
- `pomodoro:register-session` — registra sesión y emite `streak-milestone` al petWindow si hay milestone
- `pomodoro:get-stats` — retorna stats para los widgets

### Track B — I2 + W3 Quick capture + reporte semanal

#### I2 — Quick capture

Con `Cmd/Ctrl+Shift+Q` se abre un **overlay flotante** sobre la mascota
con un textarea (max 200 chars). Enter guarda, Esc cancela, click fuera
cancela. La mascota dice "¡Guardado!" como feedback.

**Persistencia**: `<userData>/quick-captures.json` (atomic write, prune a 100)

**PII redaction**: reusa `extractPII` de `pet-memories.js`. Toggle global
desde `memoriesStore.redactPII`.

**IPC**:
- `quick-capture:save` / `quick-capture:list` / `quick-capture:clear`
- `onQuickCaptureTrigger` (subscribe pattern, ya existía el cable desde v1.3.0)

**UI**:
- Overlay: `src/index.html:62-75`
- Script: `src/quick-capture-renderer.js` (134 LOC)
- Estilos: `.quick-capture-overlay`, `.quick-capture-box`

#### W3 — Reporte semanal markdown

Botón "Generar reporte semanal" en tab Pomodoro. Genera markdown con:

```markdown
# Reporte Semanal — Mascota Virtual

**Período**: 14/07 — 20/07

## Pomodoro
- Focus blocks completados: 18
- Tiempo total en focus: 7h 30m
- Por día: lun 4, mar 3, mié 2, jue 5, vie 3, sáb 1, dom 0

## Capturas
- Total: 12 ideas
- Top 5:
  - "revisar PR de Jorge antes del viernes"
  - ...

## Racha
- Actual: 7 días
- Mejor: 12 días

## Score de productividad
**75/100** — bien encaminado
```

**Score heurística**:
```
score = min(100, focusBlocks * 5 + min(20, streak * 2) + min(15, focusHours * 3))
```

**Score label**:
- 0-20: empezando
- 21-40: en ritmo
- 41-60: bien encaminado
- 61-80: muy bien
- 81-100: crack

**IPC**:
- `weekly-report:get` (con opts: weekStart='monday'|'sunday', today)

**Copy-to-clipboard** + **guardar como .md**.

**Cross-track dep**: usa `pomodoro-store` (sessions + completedDays) +
`computeStreak` + `computeLongestStreak` para el reporte completo.

### Track C — I7 + I8 Daily briefing + summary

#### I7 — Morning briefing

Al abrir la app entre las 7:00 y 12:00 (primera vez del día), la mascota
saluda con un briefing corto (≤200 chars) que incluye:
- Saludo según hora (buenos días / buen día)
- Tono cat/dog (miau / guau)
- Mensaje de bienvenida

**Persistencia**: `<userData>/daily-briefing.json` (atomic write)

**Trigger**: `app.whenReady` + 3s delay (para que la ventana esté lista)

**IPC**:
- `briefing:get-state` (consultar enabled + lastShownDate)
- `briefing:set-enabled` (toggle desde Settings)
- `briefing:show-now` (forzar desde dashboard)

**Toggle en Settings** (default ON).

#### I8 — Evening summary

Al cerrar la app después de las 18:00, la mascota muestra un resumen
del día (≤200 chars):
- Saludo (buenas noches)
- "Hoy X focus (Y min)"
- "Racha: Z días"
- "Mañana seguimos"

**Trigger**: `app.on('before-quit')` antes de clearAllTimers.

**Persistencia**: misma que briefing (1 archivo compartido).

## Decisiones técnicas

1. **Cross-track dependency** consolidada en merge final: Track B ya no usa
   dynamic require con try/catch para pomodoro-store, usa los imports directos.

2. **`computeLongestStreak`** agregado a `pomodoro-streak.js` durante el merge
   (no estaba en ninguno de los 3 tracks originales). Es helper que itera
   completedDays ordenados y devuelve la racha más larga histórica.

3. **Daily briefing automático** (Track C) usa texto simple (no stats reales)
   porque se implementó antes de que Track A y B estuvieran mergeados.
   Enriquecer con `getPomodoroStatsYesterday` + `pendingCaptures` en v1.8.0.

4. **Adaptive state en localStorage** (no en main): el contador de
   `focusBlocksCompleted` y `lastBreakWasLong` se persiste en
   `localStorage` del dashboard. Trade-off: si el usuario reinicia la app,
   el contador se resetea. El streak AUTHORITATIVO lo calcula el main con
   `computeStreak(completedDays)`.

5. **PII redaction toggle global**: memories + captures usan el mismo flag
   `memoriesStore.redactPII` (configurado en Settings). Un solo toggle
   controla ambos.

6. **Plantillas custom** extienden 4 campos (focus/break/longBreak/longBreakEvery)
   en vez de 2 (focus/break) como decía el plan original. Razón: el break
   largo es parte de la experiencia adaptativa, el usuario custom debe poder
   configurarlo. Si el user no quiere complejidad, usa una de las 3 fijas.

7. **`pet-mood` no se tocó**: la integración adaptive + mood se difiere a
   v1.8.0. Por ahora el adaptive es independiente del mood.
