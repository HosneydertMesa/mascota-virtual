## Features implementadas

### P7 — Custom name
- Input en dashboard → `Settings` → "Nombre de la mascota"
- Validación: letras, números, espacios, `. _ - '` (max 24 chars)
- Persistencia: `<userData>/pet-name.json`
- Default fallback: `Luna` (cat) o `Max` (dog)
- System prompt de la IA usa el nombre custom
- Fallback seguro: si el nombre guardado es inválido, usa Luna/Max

### M1 — Respiración
- CSS keyframe `petBreathe`: scaleY 1.00 → 1.04 → 1.00, cada 3.2s
- Aplicado al `#pet-svg-wrapper` con clase `.breathing`
- Sutil, no distrae (4% de escala)

### M3 — Pupil dilation nocturna
- Pure function `shouldPupilDilate(date)`: true entre 20:00 y 07:00 hora local
- Toggle de clase `pupils-dilated` en `<body>` cada hora
- Radio de pupila: 2.0 px (día) / 3.2 px (noche)
- Transición suave (0.6s) en el cambio

### M4 — Yawn (placeholder)
- Trigger cada 5 min en idle (mousemove resetea el timer)
- Animación CSS: scaleY + opacity, 1.6s
- Speech bubble: "¿un descansito quizás?" (placeholder — en batch 1.5 se conecta a mood system)
- En esta versión no afecta mood (es standalone)

### Archivos modificados

- `preload.js` — expone `getPetName()`, `setPetName(name)`

- `package.json` — agrega 2 archivos al `check`

### Pure functions vs renderer
`pet-micro-presence.js` se carga en `main.js` (Node) con `require`. En el renderer (browser), las mismas funciones se reimplementan minimalmente porque el renderer no tiene `require`. Tradeoff conocido: duplicación mínima de lógica, pero la API de `index.html` carga `core/pet-protocol.js` por `<script>`. En próxima iteración se puede exponer via `preload` con `contextBridge`.

## Lecciones aprendidas

### 1. Sub-agentes con `run_in_background: true` son poco confiables para features M+
El worker 2 (micro presence) terminó "succeeded" sin hacer ningún commit. Mismo síntoma que el worker 1 (mood) que también planeó sin implementar. **Causa probable**: el modelo que corre los sub-agentes tiene timeout corto o se queda sin contexto.

**Decisión**: para features M/L, mejor hacerlo en sesión activa con todo el contexto. Workers son útiles para tareas XS bien acotadas o verificación.

### 2. Worktree helper funcionó perfecto
`npm run sdlc:worktree -- add feat/b1-micro-presence` creó el directorio aislado. Sin pisarse con el repo principal. Cuando lo mergeé, ningún conflicto con otros cambios (branch huérfano). **Vale el costo de crear el helper**.

### 3. M2 (eye tracking) se quedó a medias
El JS está (initPupilPositions + handlePupilTracking + computePupilPosition), pero falta:
- Modificar los SVGs de cat.js y dog.js para tener `<circle class="pet-pupil" data-anchor-x data-anchor-y>`
- O usar los grupos `<g>` existentes de los ojos y agregar las pupilas
**Tiempo estimado**: 1-2 horas. Lo dejo para próxima sesión o batch 1.5.
