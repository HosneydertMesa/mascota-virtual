## Resumen ejecutivo

Batch 1 (v1.4.0 + v1.5.0) había dejado dos features a medias (M2 eye tracking, M4 mood-aware yawn) y un feature de UX pendiente (dashboard widget del mood). v1.5.1 cierra esos pendientes + un fix de infra que rompía el script de worktrees en Windows.

**Lo que cambia para el usuario**:
1. Los ojos de la mascota **siguen al cursor** (M2, antes fallaba silencioso).
2. El dashboard ahora tiene un **widget de estado de ánimo** con chip + 4 barras de stats.
3. Cuando la mascota está cansada, **bosteza más seguido** (M4 mood-aware).

**Lo que cambia para el dev**:
- `sdlc:worktree clean` ahora corre en Windows (antes fallaba con `grep: not found`).
- `pet-mood.js` ahora es UMD-lite (funciona en Node y browser).
- Tests: 211 → 243 (+13 SVG structure, +9 mood labels, +12 mood-aware yawn).

## Features

### M2 — eye tracking real

- `src/assets/cat.js` (idle + walk) y `src/assets/dog.js` (idle + walk): nuevo `<circle class="pet-pupil" data-anchor-x="X" data-anchor-y="Y">` dentro de cada ojo. El highlight blanco existente ahora sigue al cursor; el segundo highlight chico queda como "twinkle" fijo.
- `src/renderer.js`: corrige el divisor de viewBox en `handlePupilTracking`. Era `wrapperRect.width / 130` (incorrecto) y ahora es `wrapperRect.width / 200` (el viewBox real de los SVGs es `0 0 200 200`). Sin este fix, el cursor se mapeaba al 65% del área real y la pupila apenas se movía.
- `test/pet-assets.test.js` (nuevo, 13 tests): valida que los SVGs tengan la estructura correcta (cantidad de pupils por SVG, anchors en el centro de cada ojo, todos dentro del viewBox 0..200). Esto previene el fallo silencioso del JS si alguien rompe el SVG.

**Limitación conocida** (MINOR-1 del review): en los profile SVGs (cat/dog walk), la pupila puede sobresalir ~1.6px del ojo en dirección x porque el eye es un ellipse con rx=4. Diferido — el estado walk es transitorio (~5-10s) y el efecto es apenas perceptible.

### Dashboard widget del mood

- `preload.js`: expone `window.api.getMood()` que invoca la IPC `mood:get` (ya existía en main.js, solo faltaba exponerla).
- `src/core/pet-mood.js`: UMD-lite (CommonJS para Node, `window.PetMood` para browser). Permite que el dashboard use `deriveState()` directamente.
- `src/core/pet-mood-labels.js` (nuevo, 48 LOC): mapeo estado+petType → emoji+texto con género en español. Compartido entre browser y tests.
- `src/dashboard.html`: nuevo `<settings-group>` con un chip (estado actual con color de fondo) y 4 mini barras (energy/happiness/curiosity/hunger).
- `src/dashboard-renderer.js`: `refreshMoodWidget()` llama `getMood()` vía IPC, usa `PetMood.deriveState` y `PetMoodLabels.getMoodLabel`, actualiza el DOM. Polling cada 5s.
- `src/styles.css`: estilos del chip (azul=calm, verde=happy, violeta=sleepy, gris=sad, naranja=bored) y barras (cian/rosa/violeta/amber).
- `test/pet-mood-labels.test.js` (nuevo, 9 tests): valida que los 5 estados tengan labels en cat/dog, género correcto (cat -a, dog -o), fallback a calm.

**Limitación conocida** (MINOR-2 del review): el chip se actualiza cada 5s. Si el usuario cambia el tipo de mascota (cat↔dog) en Settings, el label del chip puede quedar desactualizado por hasta 5s. Diferido — latencia aceptable y el usuario está en la tab cuando lo cambia.

### M4 — mood-aware yawn

- `src/core/pet-micro-presence.js`: nueva pure `getYawnIntervalMs(mood)` que retorna `YAWN_INTERVAL_TIRED_MS` (2 min) si `energy < 25`, o `YAWN_INTERVAL_DEFAULT_MS` (5 min) en caso contrario. Exporta las constantes.
- `src/renderer.js`:
  - `checkYawn` ahora es `async`: hace IPC `mood:get` para saber el mood actual y elegir el intervalo. Si la IPC falla (main no listo), cae al default (5 min).
  - Reimplementación local de `getYawnIntervalMs` (mismo patrón que `computePupilPositionLocal` — el renderer no tiene `require`).
  - Las guards `idleMs < intervalMs` y `elapsed < intervalMs` se aplican con el intervalo mood-aware.
- `test/pet-micro-presence.test.js` (+12 tests): cubre `getYawnIntervalMs` (energy 0/24/25/50, mood null/undefined/energy no-numero) y `shouldYawn` con interval corto (2 min).

## Bug fixes

### `sdlc:worktree clean` portable

- `scripts/sdlc-worktree.js`:
  - Reemplaza `git branch --merged main | grep -w "<branch>"` por parseo en Node (Set con trim). `grep` no existe en Windows.
  - `mainBranch` detection robusta: parsea `git symbolic-ref refs/remotes/origin/HEAD` con regex, valida que sea `main`/`master`, sino cae a `git rev-parse --abbrev-ref HEAD`. Antes, si el upstream HEAD apuntaba a un branch stale (ej: una feature branch), la comparación `branch === mainBranch` fallaba siempre.
  - `2>/dev/null` → `execSync` con `stdio: 'ignore'` (cross-platform).

Probado en Windows: `node scripts/sdlc-worktree.js clean` ahora corre sin error.
