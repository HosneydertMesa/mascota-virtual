## Features

### P3 — Recuerdos persistentes (Track A)

Ver `v1.6.0-track-a-changelog-2026-07-21.md` para detalle completo.

Resumen:
- 50 recuerdos max en `<userData>/pet-memories.json`
- Memory extractor IA (1 llamada extra por mensaje, ~200 tokens)
- PII redaction ON por default (email, phone, credit card)
- Dedup Jaccard 0.7, prune al límite
- Top 5 relevantes al system prompt via TF bag-of-words
- Dashboard tab "Recuerdos" con lista + clear + toggle PII
- 96 tests (38 pure + 37 store + 21 extractor)

### A3 — Idle detection (Track B)

**Concepto**:
- Cada 60s, main consulta `powerMonitor.getSystemIdleTime()` (built-in Electron, cross-platform, sin permisos)
- Si el SO lleva 10+ min sin input Y no estamos en cooldown (5 min), sugiere un break
- Tip: "Llevas X min sin actividad. ¿Un break?" como speech bubble en la mascota

**Archivos nuevos**:
- `src/core/context-awareness.js` (161 LOC): pure functions — isSystemIdle, shouldSuggestBreak, formatIdleTime
- `src/services/idle-monitor.js` (119 LOC): bridge a powerMonitor con tick, cooldown, try/catch defensivo

**Archivos modificados**:
- `main.js`: importa createIdleMonitor, en app.whenReady() crea el monitor, en before-quit() detach
- `src/renderer.js`: handle del event='idle-break' en onSystemEvent, muestra speech bubble

### A4 — Typing rate Do Not Disturb (Track B)

**Concepto**:
- Dashboard mide keystrokes en el chat input (event='input')
- Rolling window 5 min, chequea cada 10s
- Si > 80 WPM sostenido por 2+ min: activa DND via IPC `dnd:update`
- Si typing < 60 WPM por 30s: sale de DND
- Mientras DND: autonomous cycle en main se saltea (no interrumpe al usuario)

**Archivos modificados**:
- `main.js`: variable `isDoNotDisturb`, IPC handler `dnd:update`, startAutonomousCycle respeta el flag
- `preload.js`: expone `setDoNotDisturb(isActive)`
- `src/dashboard.html`: script de context-awareness.js
- `src/dashboard-renderer.js`: listener input, rolling buffer, maybeCheckDnd, beforeunload cleanup
- `src/renderer.js` (ya cubierto por A3)

**Tests**: 44 nuevos (31 context-awareness + 13 idle-monitor)

## Decisiones de diseño

| Decisión | Por qué |
|---|---|
| A3 usa `powerMonitor.getSystemIdleTime()` | Built-in Electron, cross-platform, NO requiere permisos OS |
| A4 mide typing en el chat (no system-wide) | Más simple, no requiere permisos, cubre 90% del caso de uso |
| A3 cooldown 5 min | Evita spam de tips si el usuario está realmente ausente |
| A4 DND threshold 80 WPM sostenido 2 min | Generoso (no interrumpe sesiones cortas de programación) |
| A4 DND exit 60 WPM por 30s | Recovery rápido cuando el usuario para de escribir |
| A3 system-wide idle (no solo de la app) | Es lo que el usuario quiere — la mascota "se preocupa" |
| A4 keyboard events via 'input' (no 'keydown') | Cuenta paste, deletion, etc consistentemente |

---

## Limitaciones conocidas

1. **A3 mide idle del SO entero** — si el usuario está en otra app por 10 min, también sugiere break. Aceptable.
2. **A4 solo mide typing en el chat** — typing en IDE no activa DND. Out of scope para v1.
3. **A4 cuenta backspace como keystroke** — typing inflado al borrar. Documentado en MINOR-4.
4. **A3 depende de `powerMonitor`** — en algunos Linux sin systemd-logind puede no funcionar. Defensivo: no-op si no está disponible.
5. **A4 persiste flag DND solo en memoria** — al cerrar dashboard, beforeunload limpia. Al reiniciar app, DND está off.
